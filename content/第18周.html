第18周

課程總結

以下是我從w2~w17這期間內的所有筆記內容

暫存修改
git add .

提交修改
git commit -m "更新!"

檢查工作目錄狀態
git status

推送到遠端
git push origin main

用cms查詢網路
cms

開啟組別網址
https://127.0.0.1:9443
(只有本地端開起來後才能執行)

進入後輸入admin登入

在git上開啟修改簡報
notepad Y:\tmp\work\respysive_slide\readme_example.py

完成後執行
python Y:\tmp\work\respysive_slide\readme_example.py

----------------------------------------------
建立分支
git branch 學號

切換分支
git checkout 學號

暫存修改
git add .

檢查工作目錄狀態
git status

提交修改
git commit -m "更新!"

推送到自己分支的遠端
git push --set-upstream origin 學號

如果要合併就照以下步驟"一行一行"的輸入CMS
git checkout main
git pull origin main --rebase
git merge 41423230
git push origin main

分隔線以下是在教怎麼建立分支(記住是在自己的電腦上打這些指令，就是在CMS上做啦~~!)，這樣大家建立好分支後就不會直接覆蓋掉主分支的檔案，上傳的檔案也只會在自己的分支上，之後簡報我再引用分支上的內容就好了。分隔線以下的學號請改成自己的學號

-----------------------------------------------------------
開啟本地端AI
ollama run
檢視模型種類
ollama list
儲存對話
/save
---------------------------------------------------------
讓程式可以在伺服器上永久執行
先切換至檔案存在的資料夾
由於只有使用者權限因此使用nohup的方式執行檔案
nohup python3 run_server.py > server.log 2>&1 &
確認執行
tail -f server.log
ctrl+c切回cmd對話視窗
用查詢背景正在執行的PID
ps aux | grep run_server.py
用 pkill一次清掉所有process
pkill -f run_server.py
查誰佔著 8023
lsof -i
----------------------------------------------------
遠端伺服器執行
輸入帳號
ssh a35@s5.eng.nfu.edu.tw
切換至1-1目錄
cd 1-1
切換至1-1下的1資料夾
cd 1
用python3執server.py檔案
python3 server.py
開啟網址(組員只需網址即可上去編輯)
https://s5.eng.nfu.edu.tw:9035
其餘以此類推
